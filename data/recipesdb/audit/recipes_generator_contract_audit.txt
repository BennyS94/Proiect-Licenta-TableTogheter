8. Recommended generator-ready eligibility rule
Recommended base rule for generator v1:
- recipes.scope_status = pilot_validated
- recipes.is_active = 1
- recipes.has_ingredients_parsed = 1
- recipe_nutrition_cache.cache_status = partial_from_mapped_ingredients

Result on current pilot:
- generator-ready base pool = 76 recipes

Useful stricter subset if early nutrition scoring needs better coverage:
- base rule above
- plus recipe_nutrition_cache.mapped_weight_ratio >= 0.20

Result on current pilot:
- nutrition-preferred subset = 43 recipes

9. First generator-related implementation recommendation
- First generator adaptation should consume recipes + recipe_ingredients as the primary truth.
- recipe_nutrition_cache should be treated as an optional soft layer for filtering/scoring, not as a hard dependency for every recipe row.
- Start with a generator read adapter over the 76 base-eligible recipes before adding any broader scoring or substitution logic.
